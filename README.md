# Video Transcriber

Автоматическая транскрибация видео с использованием AI (Whisper от Groq). Превращает любое видео в текстовую расшифровку с таймкодами.

## Особенности

- **Автоматизация** - просто запустите скрипт с путем к видео
- **Пакетная обработка** - обрабатывайте целые папки с видео
- **Умный пропуск** - не обрабатывает повторно уже транскрибированные видео
- **Высокая точность** - использует Whisper от Groq
- **Автопереключение моделей** - при rate limit автоматически переключается между моделями
- **Retry логика** - автоматические повторы при ошибках API
- **Таймкоды** - получайте JSON с точными временными метками
- **Мультиязычность** - поддержка русского, английского и других языков

## Поддерживаемые форматы

`.mp4`, `.avi`, `.mov`, `.mkv`, `.webm`, `.flv`, `.wmv`

## Требования

- **Node.js** v18+ ([скачать](https://nodejs.org/))
- **FFmpeg** ([инструкция по установке](#установка-ffmpeg))
- **Groq API Key** (бесплатно на [console.groq.com](https://console.groq.com/keys))

## Быстрый старт

### 1. Клонировать репозиторий

```bash
git clone https://github.com/yourusername/video-transcriber.git
cd video-transcriber
```

### 2. Установить зависимости

```bash
npm install
```

### 3. Настроить API ключ

Создайте файл `.env` в корне проекта:

```env
GROQ_API_KEY=your_groq_api_key_here
```

Получить ключ: https://console.groq.com/keys

### 4. Установить FFmpeg

#### Windows

```powershell
# Через winget
winget install Gyan.FFmpeg
```

После установки найдите путь к ffmpeg:
```powershell
where ffmpeg
```

И обновите путь в `transcribe.js` (строка FFMPEG_PATH).

#### macOS

```bash
brew install ffmpeg
```

#### Linux

```bash
# Ubuntu/Debian
sudo apt install ffmpeg
```

### 5. Запустить транскрибацию

```bash
# Одно видео
node transcribe.js video.mp4

# Вся папка с видео
node transcribe.js ./videos
```

## Конфигурация

Отредактируйте объект `CONFIG` в `transcribe.js`:

```javascript
const CONFIG = {
  CHUNK_DURATION: 1200,        // Длина частей (секунды) - 20 мин
  MODELS: ["whisper-large-v3-turbo", "whisper-large-v3"], // Модели для переключения
  LANGUAGE: "ru",              // Язык: ru, en, auto
  RATE_LIMIT_DELAY: 15000,     // Пауза между запросами (мс)
};
```

## Лимиты Groq API

Бесплатный план имеет следующие лимиты для каждой модели:

| Лимит | Значение |
|-------|----------|
| Запросов в минуту | 20 |
| Запросов в день | 2000 |
| Аудио секунд в час | 7200 (120 мин) |
| Аудио секунд в день | 28800 (480 мин) |

**Важно**: Лимиты считаются отдельно для каждой модели! Скрипт использует две модели (`whisper-large-v3-turbo` и `whisper-large-v3`), что удваивает доступный лимит до **240 минут аудио в час**.

## Решение проблем

### 403 Forbidden

**Причины:**
- Неверный API ключ
- Ключ деактивирован или истёк
- Региональные ограничения (попробуйте VPN)
- Новый ключ ещё не активировался (подождите 1-2 минуты)

**Решение:**
1. Проверьте ключ на https://console.groq.com/keys
2. Создайте новый ключ
3. Подождите пару минут после создания нового ключа
4. Попробуйте с VPN (США/Европа)

### 429 Rate Limit

**Причина:** Превышен лимит запросов или аудио секунд.

**Решение:**
- Скрипт автоматически переключится на другую модель
- Если обе модели в rate limit - подождите до следующего часа
- Увеличьте `RATE_LIMIT_DELAY` в конфигурации

### 401 Invalid API Key

**Причина:** Ключ недействителен.

**Решение:**
1. Проверьте что ключ в `.env` скопирован полностью
2. Убедитесь что нет лишних пробелов
3. Создайте новый ключ на https://console.groq.com/keys

### Пустые файлы транскрипции

**Причина:** API вернул ошибку, но скрипт не обработал её корректно.

**Решение:**
- Проверьте консоль на наличие ошибок HTTP
- Убедитесь что API ключ работает
- Попробуйте запустить снова

## Как это работает

```
Видео → FFmpeg (извлечение аудио) → Нарезка на части по 20 мин → Groq Whisper API → Текст + JSON
```

1. **Извлечение аудио** - FFmpeg конвертирует видео в MP3 (моно, 16kHz)
2. **Разделение** - Аудио режется на 20-минутные части
3. **Транскрибация** - Каждая часть отправляется в Groq API с retry логикой
4. **Автопереключение** - При rate limit переключается на другую модель
5. **Сохранение** - Результаты объединяются в transcript.txt и transcript.json

## Производительность

- **52-минутное видео** → ~2-3 минуты обработки
- **Пауза между частями**: 15 сек (для избежания rate limit)
- **Retry**: до 2 попыток на модель, потом переключение
